### 問1: フレームの大きさ

フレームの構成：
- 衛星回線フレームヘッダ: 20バイト
- IPヘッダ: 20バイト
- TCPヘッダ: 20バイト
- データ: 740バイト

**フレームの大きさ = 20 + 20 + 20 + 740 = 800バイト**

### 問2: フレームの伝送遅延

フレームサイズ: 800バイト = 800 × 8 = 6400ビット
回線速度: 128 kbit/sec = 128000 bit/sec

**伝送遅延 = 6400 ÷ 128000 = 0.05秒 (50ミリ秒)**

### 問3: ストップアンドウェイトARQの最大スループット

1サイクル時間 = 伝送遅延 + 往復伝搬遅延
= 0.05 + 0.295 × 2 = 0.64秒

実効データ量: 740バイト = 740 × 8 = 5920ビット

**最大スループット = 5920 ÷ 0.64 = 9.25 kbit/sec**

### 問4: ウィンドウ制御による連続ARQの最大スループット（ウィンドウごとに確認応答）

ウィンドウごとに確認応答する場合、ウィンドウサイズ分のフレームを送信してからACKを待ちます。

1サイクル時間 = (ウィンドウサイズ × 伝送遅延) + 往復伝搬遅延

各ウィンドウサイズでの結果：
- **ウィンドウサイズ1**: 1サイクル時間 = 0.05 + 0.59 = 0.64秒
  スループット = (1 × 5920) ÷ 0.64 = **9.25 kbit/sec**

- **ウィンドウサイズ5**: 1サイクル時間 = 0.25 + 0.59 = 0.84秒
  スループット = (5 × 5920) ÷ 0.84 = **35.24 kbit/sec**

- **ウィンドウサイズ10**: 1サイクル時間 = 0.5 + 0.59 = 1.09秒
  スループット = (10 × 5920) ÷ 1.09 = **54.31 kbit/sec**

- **ウィンドウサイズ100**: 1サイクル時間 = 5.0 + 0.59 = 5.59秒
  スループット = (100 × 5920) ÷ 5.59 = **105.9 kbit/sec**

### 問5: スライディングウィンドウ制御の最大スループット

スライディングウィンドウでは、ACKが戻ってくる前に次のフレームを連続的に送信できます。

理論的最大スループット = (実効データ ÷ フレーム全体) × 回線速度
= (740 ÷ 800) × 128 = 118.4 kbit/sec

パイプライン容量 = (往復遅延時間 ÷ 1フレーム伝送時間) + 1
= (0.59 ÷ 0.05) + 1 = 約13フレーム

各ウィンドウサイズでの結果：
- **ウィンドウサイズ1**: 効率 = min(1, 13) = 1 → **9.25 kbit/sec**
- **ウィンドウサイズ5**: 効率 = min(5, 13) = 5 → **59.2 kbit/sec**  
- **ウィンドウサイズ10**: 効率 = min(10, 13) = 10 → **118.4 kbit/sec**
- **ウィンドウサイズ100**: 効率 = min(100, 13) = 13 → **118.4 kbit/sec**
